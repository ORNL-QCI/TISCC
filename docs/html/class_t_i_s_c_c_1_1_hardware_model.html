Detailed Description

Defines a set of native hardware operations and provides an interface for constructing circuits out of them.

Constructor & Destructor Documentation

◆ HardwareModel()

TISCC::HardwareModel::HardwareModel ( )
explicit

Constructs a HardwareModel object with default trap width and initializes a map of trapped-ion operations to their default durations.

Trap width, available operations, and their durations are hard-coded but can be straightforwardly adapted within the code.

◆ shift_left()

double TISCC::HardwareModel::shift_left ( const std::set< unsigned int > &  qubits,
unsigned int  patch_cols,
GridManager &  grid,
std::vector< HW_Instruction > &  hw_master,
double  time 
) const

Shifts a group of data qubits one column left-ward on the hardware grid.

This function is responsible for moving a set of qubits one column left-ward on the hardware grid. It depends on the details of our chosen hardware architecture specified in GridManager. It assumes that all input qubits are on 'O' qsites and that there are no occupied 'M' sites along the relevant paths. It first moves the input qubits out of the way (adjacent to the south-west 'O' site), then shuttles the strip of qsites (if occupied) from the left-ward column to the other end of the tile. Finally, it moves the input qubits to their final positions.

Parameters
qsitesA set of qubits to be shifted left-ward.
patch_colsThe number of columns in the surface code patch.
gridThe hardware grid on which to perform qubit movements.
hw_masterThe master circuit to which the Move operations are added.
timeThe current time in microseconds.
Returns
The updated time after performing the Move operations.
